家のリノベを検討していて、間取りを自分でいじってみたくなった。で、間取りソフトをいくつか触ってみたんだけど、どれもプロ向けで難しすぎる。寸法の入れ方、レイヤー、CAD的な概念…。やりたいのは「この部屋もう少し広げたら？」「ここに収納足したら？」を感覚で試すことなのに、その手前の学習コストが高すぎて手が止まる。

なので、自分用に作った。ブラウザで動く2Dの間取りエディター、**Madori Renovation Editor**。

- 公開URL: https://madori-renovation-editor.vercel.app/
- GitHub: https://github.com/crazyathlete220-stack/madori-renovation-editor

この記事は宣伝というより制作ログで、何を作って、どこで判断を変えて、どこで詰まったかを残しておく。

## 何を作ったか

ブラウザで開いてすぐ触れる2D間取りエディター。配置できるのは部屋・壁・ドア・窓・家具・設備・収納・外構パーツ。キャンバス上でドラッグして動かして、つまんでリサイズして、右のパネルで座標や寸法を数値でも微調整できる。

実際の家づくりは平屋とは限らないので、**階ごとの編集**にも対応した。1階・2階…とタブで切り替えて、階の追加もできる。敷地サイズも変えられるので、自分の土地に合わせて引いていける。保存は今のところ `localStorage` で、リロードしても消えない。

データは画像じゃなくてJSONで持っている。`plan.meta`（案件情報）の下に `plan.floors`（階別データ）があって、その中に `zone`（部屋）/ `wall`（壁）/ `opening`（ドア・窓）/ `item`（家具・設備・収納・外構）がぶら下がる構造。画像で持つと後から編集も再利用もできないので、最初から構造化データにしておいた。これは後々AIに読ませたり出力形式を増やしたりするときに効いてくるはず。

## 「AI相談は後」と決めた

最初、AIに「家族構成と予算を入れたら間取りを提案してくれる」みたいな機能から作ろうとしていた。でも途中でやめた。

理由は、**人が直感的に触れるエディターが無いと、AIが何かを提案してもその先に進めない**から。提案された間取りを「ここだけ直したい」と思ったときに、結局手で調整できる場所が要る。だったら順番は逆で、まず人が気持ちよく編集できる土台を作って、AI相談はその上に後から乗せるべきだと考え直した。

なので今回はAI機能をきっぱり後工程に切り出して、編集体験そのものに集中した。

## プロ向けの項目を、削る判断

作っているうちに、つい「業務でも使えるように」と項目を増やしてしまった。予算、住所、入居予定日、家族構成、提案先、工事区分、段階、構造…。気づいたら、自分が最初に「難しすぎる」と思って避けたツールに近づいていた。

ここで方針を戻した。**これは個人が間取りで遊ぶためのもの**で、提案業務の管理ツールじゃない。上に挙げた項目は全部外した。残したのは案件名・階・敷地・面積・メモくらいの最小限。

UXも業務ツール感を抜く方向に振った。

- メニューは折りたためるようにして、最初の画面をすっきりさせた
- 削除はゴミ箱アイコンで分かりやすく
- 操作ミスはUndo（一つ戻る）で気軽に取り消せる
- 壁の厚みは固定にして、ユーザーが気にしなくていいようにした

特に効いたのが**スナップ**で、移動やリサイズが一定グリッド（内部的には `SNAP_SIZE = 20`）に吸い付く。ドラッグ中はSNAP表示や演出を出して、ゲームのビルドモードっぽい操作感に寄せた。「数値を正確に入れる」じゃなくて「だいたいの位置に置いたら勝手に整う」感覚にしたかった。

## 技術構成

シンプルに React + TypeScript + Vite、ホスティングは Vercel。状態管理ライブラリは入れず、Reactのstateで完結させている。キャンバスはSVGで描画。今のところ実体はほぼ一枚の大きめなコンポーネントで、間取りのJSONを状態として持ち、それをSVGに描いて、ドラッグ操作で書き換える、という素直な作り。

## 詰まったところ

**壁を広げたあと元に戻らない問題。** 壁を普通の矩形と同じリサイズロジックで動かしていたら、一度伸ばした壁が縮められなくなった。壁は「長さ」と「厚み」で挙動が違うので、壁専用のリサイズルールに分離して解決した。厚みは固定（`MIN_WALL_THICKNESS = 8`）にして、基本は長さ方向だけ伸縮するようにしている。

**タイトルと上部メトリクスが重なる問題。** 階の面積や要素数を出す上部の表示と画面タイトルがレイアウトでぶつかった。地味だけどこういうのを潰さないと「ちゃんとしてない感」が出る。

**項目を入れすぎて個人向けじゃなくなりかけたこと。** これは技術というより設計の詰まり。前述の通り、足すより削るほうが難しかった。

Undoは操作のたびにスナップショットを積む方式で実装した。ドラッグ開始時に一度だけ状態を捕まえて履歴に入れる、という形にして、移動中に履歴が大量に増えないようにしている。

## これからやりたいこと

- もっとゲームっぽい吸着感（今のスナップをさらに気持ちよく）
- 置ける場所のハイライト（ここに収納が入る、みたいな視覚フィードバック）
- PNG / PDF 出力
- AI相談機能（ようやくここで乗せる）
- クラウド保存と共有リンク

順番としては、まず人が触って楽しい状態を固めて、そのあとに出力・共有・AIを足していく。最初に決めた「人の操作感が先、AIは後」という方針はこの先も崩さないつもり。
